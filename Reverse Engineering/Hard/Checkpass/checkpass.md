# checkpass

## Information
- Category: RE
- Level: Hard

## Description

What is the password?

## Hint
How is the password being checked?

## Solution
### What we got from the problem ?
- The problem gives us a ELF 64-bit LSB file. 
- Try to run the file by using `./checkpass <password>` base on it usage, with the input is `abc`, it returns `Invalid Length`. So we observed that, this checker require a target of password's length to do the checking process.

    ![Check]()

### Analyzing
- We will use IDA to decompile the binary, and checking the `main`, we can see that it contains only 2 functions, which is `sub_5960()` and `sub_226C0`:

    ```c
    int __fastcall main(int a1, char **a2, char **a3)
    {
        void (__fastcall __noreturn *v4)(); // [rsp+0h] [rbp-8h] BYREF

        v4 = sub_5960;
        return sub_226C0(&v4, &off_2482A8, a1, a2);
    }
    ```
- Try to check `sub_5960()` and `sub_226C0`, respectively:

    - First, about `sub_5960()`, this function will get the input and check the password's length. We observed that, the valid password's length is 41:
        ```c
        if ( *(_QWORD *)(v83[0] + 40LL) == 41 )
        ```
    - Moreover, this code will check the format of input. Looking closely, it is the flag format. So the correct password is the flag of this problem.:
        ```c
        if (*(_QWORD *)v0 == 0x7B4654436F636970LL)  // Kiểm tra prefix "picoCTF{"
        if (*v1 == 125)  // Kiểm tra suffix "}" (ký tự '}' = 125 ASCII)
        ```
    - Looking closely in this function, we can see `sub_54E0()` is called 4 times. Checking the `sub_54E0`, it is a **shuffle function**, with 2 steps:

        - Step 1: Substitution. With `byte_39560` is a S-Box (Substituon Box) and `a3` is a selection value in range [0..3], which means select 1 of 4 tables, each table's size is 256 bytes. For each input byte `a2[i]`, it will replace by `byte_39560[v3 + a2[i]]`. The formula is `v6[i] = S-Box[round][input[i]]`. 
            ```c
            v3 = a3 << 8;  // a3 * 256 - chọn bảng thay thế dựa trên round (0-3)
            v6[i] = byte_39560[v3 + a2[i]];  // với i = 0..31
            ```
        - Step 2: Permutation with `asc_39970` - the permutation table. The price at `asc_39970[round * 256 + out_pos * 8]` will display which byte is taken from `v6[i]`.
            ```c
            *(_BYTE *)(a1 + 8) = v6[ asc_39970[v3 + 64] ];   // output[8]  = v6[perm[8]]
            *(_BYTE *)(a1 + 25) = v6[ asc_39970[v3 + 200] ]; // output[25] = v6[perm[25]]
            // ... tiếp tục cho tất cả 32 vị trí
            ```
    - So, this is overall how the input is encrypted:
    ```
    Input (32 bytes) 
        ↓
    ┌─────────────────────────────────────┐
    │  SUBSTITUTION (S-Box lookup)        │
    │  v6[i] = SBOX[round][input[i]]      │
    └─────────────────────────────────────┘
        ↓
    ┌─────────────────────────────────────┐
    │  PERMUTATION (Reorder positions)    │
    │  output[i] = v6[PERM[round][i]]     │
    └─────────────────────────────────────┘
        ↓
    Output (32 bytes)
    ```

    - The `sub_226C0` is just the init Rust runtime function.

### How to get flag ?
- First, we need to dump 2 tables `byte_39560` and `asc_39970` from binary file to build the reverse table. This is the `idc` code to do it:

    ```idc
    #include <idc.idc>

    static main() {
        auto i;
        auto addr;
        
        // Print pbox
        Message("pbox = [");
        addr = 0x39970;
        for (i = 0; i < 1024; i++) {
            Message("%d", Byte(addr + i));
            if (i < 1023) Message(", ");
        }
        Message("]\n");
        
        // Print expected
        Message("expected = [");
        addr = 0x39D95;
        for (i = 0; i < 32; i++) {
            Message("%d", Byte(addr + i));
            if (i < 31) Message(", ");
        }
        Message("]\n");
    }
    ```
- After dumping, we can see that:

    - About S-Box, 1024 bytes is divided into 4 tables (with 256 bytes in a table):
        | Round | Offset | Address |
        | :--: | :--: | :--: |
        | 0	| 0x000	| 0x39560 - 0x3965F |
        | 1	| 0x100	| 0x39660 - 0x3975F |
        | 2	| 0x200	| 0x39760 - 0x3985F |
        | 3	| 0x300	| 0x39860 - 0x3995F |
    - About the P-table, it is read base om `QWORD` at offset in the below table, with each 8 bytes for an output:
        | Offset | Value |
        | :--: | :--: |
        | 0 | 0 |
        | 8 | 1 |
        | 16 | 2 |
        | ... | ... |
- To get the flag, we need to build Inverse Algorithm for S-Box and P-table. Moreover, because of 4 round shuffing, we need to reverse in 4 rounds to get the final flag.

def parse_pbox(pbox_raw):
    """Parse P-Box: mỗi 8 bytes = 1 QWORD index"""
    pbox = []
    for round in range(4):
        round_pbox = []
        for i in range(32):
            offset = round * 256 + i * 8
            val = int.from_bytes(bytes(pbox_raw[offset:offset+8]), 'little')
            round_pbox.append(val)
        pbox.append(round_pbox)
    return pbox

def build_inverse_sbox(sbox):
    """Xây dựng S-Box nghịch đảo"""
    inv_sbox = []
    for round in range(4):
        inv = [0] * 256
        for i in range(256):
            inv[sbox[round * 256 + i]] = i
        inv_sbox.append(inv)
    return inv_sbox

def forward_round(data, round, sbox_table, pbox):
    """Thực hiện một round forward để verify"""
    # Bước 1: Substitution
    v6 = bytearray(32)
    for i in range(32):
        v6[i] = sbox_table[round * 256 + data[i]]
    
    # Bước 2: Permutation - output[out_pos] = v6[pbox[out_pos]]
    output = bytearray(32)
    for out_pos in range(32):
        in_pos = pbox[round][out_pos]
        output[out_pos] = v6[in_pos]
